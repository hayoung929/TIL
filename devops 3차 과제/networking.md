# Networking

## 1. IPv4, IPv6

- IPv4 : 32비트 주소 체계, 약 43억개 주소 (예: 192.168.0.1)
- IPv6 : 128비트 주소 체계, 사실상 무한 (예: 2001:0db8::1)
- IPv4는 주소 고갈 문제로 IPv6로 전환 중
- IPv6는 IPSec 기본 내장, 브로드캐스트 미지원

## 2. ARP / RARP

- ARP : IP 주소 → MAC 주소 변환
- RARP : MAC 주소 → IP 주소 변환 (현재는 DHCP로 대체)
- ARP는 브로드캐스트로 MAC 주소를 물어보고, 응답받아 캐시에 저장

## 3. 3-Way Handshake

- TCP 연결을 수립하는 과정

1. Client → Server : SYN (연결 요청)
2. Server → Client : SYN + ACK (요청 수락)
3. Client → Server : ACK (확인 응답)

## 4. 4-Way Handshake

- TCP 연결을 종료하는 과정

1. Client → Server : FIN (종료 요청)
2. Server → Client : ACK (확인, 아직 데이터 전송 가능)
3. Server → Client : FIN (서버 종료 준비 완료)
4. Client → Server : ACK (최종 확인 후 종료)

### 3-Way vs 4-Way 비교

- 목적 : 3-Way는 연결 수립 / 4-Way는 연결 종료
- 단계 : 3-Way는 3단계 / 4-Way는 4단계
- 4단계인 이유 : 서버가 FIN을 받아도 남은 데이터를 모두 보내야 해서 ACK와 FIN을 분리 전송
- 4-Way에만 CLOSE_WAIT, TIME_WAIT 상태 존재

## 5. 브로드캐스트, 유니캐스트, 애니캐스트

- 유니캐스트 : 1대1 통신 (특정 단일 목적지)
- 브로드캐스트 : 1대전체 통신 (같은 네트워크 내 모든 호스트)
- 멀티캐스트 : 1대그룹 통신 (특정 그룹에게만)
- 애니캐스트 : 가장 가까운 1개 노드에게 전송 (DNS, CDN에서 사용)

## 6. TCP / UDP

- TCP : 연결 지향, 신뢰성 보장, 순서 보장, 느림 (HTTP, FTP, SSH)
- UDP : 비연결, 신뢰성 없음, 빠름 (DNS, 스트리밍, 게임, VoIP)

## 7. OSI 7 Layer

1. 물리 계층 : 비트 전송, 케이블, 허브
2. 데이터링크 계층 : MAC 주소, 프레임, 스위치, 브리지
3. 네트워크 계층 : IP 주소, 패킷, 라우터
4. 전송 계층 : TCP/UDP, 포트 번호, 세그먼트
5. 세션 계층 : 세션 수립/유지/종료
6. 표현 계층 : 암호화, 압축, 인코딩 (SSL/TLS)
7. 응용 계층 : 사용자 인터페이스, HTTP, FTP, DNS

## 8. 브리지

- 2계층 (데이터링크) 장비
- MAC 주소를 기반으로 프레임 전달
- 두 네트워크 세그먼트를 연결
- MAC 주소 테이블을 학습해서 불필요한 트래픽 차단
- 현재는 스위치로 대체됨

## 9. 스위치

- 2계층 (데이터링크) 장비
- MAC 주소 테이블 기반으로 목적지 포트에만 프레임 전달
- 브리지보다 포트 수 많고 속도 빠름
- 포트마다 독립된 콜리전 도메인 → 충돌 없음
- 전이중(Full-Duplex) 통신 지원
